## 製造業向け製版システム Web アプリケーション

### 概要

#### ドメイン

海外拠点を含む複数工場を持つ製造企業向けの、社内生産管理・帳票出力システム。

#### 目的

工場ごとに分散していた生産管理・各種帳票作成のワークフローを Web 化し、  
入力・承認・帳票出力までを一元管理できるようにすること。

#### 構成

Java ＋ JavaScript ＋ MySQL ＋ MyBatis  
＋ AWS EC2 ＋ AWS RDS ＋ AWS S3

### 案件情報

期間：2023/08 〜 2024/11  

体制：PM 1名、PL 2名、SE 3〜4名からなる 4〜7名の受託開発チーム（時期により変動）。  

自分の役割：帳票出力機能の詳細設計・実装・テストおよび運用保守を担当。

### 担当業務

- Java（JDK 17）を用いた帳票出力機能（PDF 発注書）の新規開発  
  - Service 層／Repository 層の実装、画面との連携処理、SQL 実行ロジックの実装  
  - XML ベースの SQL 定義（MyBatis）を用いたデータ取得処理と帳票出力処理の実装
- 既存ソースコードの調査・流用により、レイアウト設計および帳票仕様の整理・調整を実施
- Redmine を用いた課題管理に基づく不具合修正、および単体〜結合テストの実施

### 技術スタック

**言語・FW:** Java, JavaScript, MySQL, XML, MyBatis  

**インフラ:** AWS EC2, AWS RDS, AWS S3  

**ツール類:** A5:SQL Mk-2, TortoiseSVN, WinMerge, LibreOffice, STS4, Redmine, Everything

### Key ADR

本PJでは、基本方針や設計方針はすでに上流工程で決定されており、  
自分の担当はテスト設計・実装・エビデンス作成が中心だった。

そのため、このPJで自分の裁量で選択した技術的な意思決定はほとんどなく、  
Key ADR は設定せず、「振り返り」で得られた学びのみを整理する。

### 振り返り（学び・次に活かしたいこと）

- 上流で設計が固まった状態から、実装とレビュー対応を繰り返す形となり、方針そのものを変える余地はほとんどなかった。  
- 何度も差し戻しを受ける中で、本来はレビュー前に自分で観点を整理しておくことが望ましいと理解しつつも、この時点では十分に実践できておらず、のちの PHP プロジェクトでようやくセルフチェックを意識して行うようになった。
